Some calamities are rendered procedurally, meaning they are built up of multiple models and all tacked together via code in the renderer. These calamities do not have one "full" model, but instead a bunch of isolated segments. Some calamity model folders offer a "wiki render" variant of the model featuring an attempt to stitch together the model segments into one coherent model, however the actual accuracy of these models are... not insured.

Currently the calamities that utilize these multi-model make ups include:

War-torn Sieger adaptation:
- Base model
- "Arrows" overlay

Hohlfresser:
- Head model
- 3 unique segment models
- Tail model
- (wiki model included)
NOTE: due to how the rendered segments are manually scaled down via code, in order to access the proper model state seen in the wiki, you need to open the animation tab and select the relevant animation. A wiki model for the adapted variant is NOT supplied at the current moment.

Grakensenker:
- Body model (base, adapted)
- 12 unique "segment" models
- 2 hand models
- foot model
- (wiki model included)

Leviathan:
- Body model
- Middle segment model
- Tail model
- Missing: Leg model, wiki model

Do also note that the "costumes" some calamities don during specific dates (E.G. Xmas) are usually handled via additional model layers, akin to the War-torn Sieger adaptation arrows overlay
(some of these costumes are included, however no guarantees are made. "Wiki" models are not supplied for these costumes)

Other calamities will use this rendering format in the future, and so this .txt doc will be updated accordingly.
